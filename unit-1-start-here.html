Unit 1: Start Here

First steps to get familiar with Java compilation and command-line tools.

  1. Open a command-line window.
  2. Change to the start-here directory.

  3. See which version of Java is installed on your system:

    java -version

    This displays the version of Java that is found on your system's PATH.

  4. Start the Java runtime:

    java

    This starts the Java runtime. When you don't provide any command-line arguments, it displays the available command-line arguments by default.

  5. List the contents of the directory. If you are on a *nix system, enter

    ls -l

    If you are on a Windows system, enter

    dir

    Notice that the directory contains a Java source file named CompileMe.java. The names of Java source files always end with .java.

    List the contents of the source file. If you are on a *nix system, enter

    cat CompileMe.java

    If you are on a Windows system, enter

    type CompileMe.java

    You should see something like this: 

    public class CompileMe {
    public String thisClassName() {
        return this.getClass().getName();
    }
}

    From this, you can surmise that a Java source file contains a class declaration, and the contents of the class are enclosed within a pair of curly braces. This class contains a single method. You can surmise from the example that the contents of a method are enclosed within a pair of curly braces.

    You might guess that Java uses curly braces to delimit blocks of code. This would be a good guess.

    Regarding naming conventions, the example suggests that a Java class name begins with an upper-case letter and follows camel-case capitalization. The source file name is exactly the same as the class name it contains, except for the suffix .java.

    There are a few other things in the source file that you might not understand. Don't worry, everything will be explained in due time.

  6. Now compile the source file:

    javac CompileMe.java

    This command runs the Java compiler. You pass the names of one or more Java source files as command-line arguments. You might have noticed that the compiler did not write any output to the console. This is the defined behavior of the Java compiler when no compilation errors occur.

    List the contents of the directory again, and you will see a new file there named CompileMe.class. From this, you might surmise that the names of Java object files end with .class.

    Java class files do not contain native object code for the hardware platform. They contain byte codes. Byte codes are object code for the Java Virtual Machine (JVM). A JVM is a software implementation of a machine architecture, much like the hardware architecture of an Intel chipset or the IBM z/Architecture Central Processor. The same "virtual" architecture is implemented in native code for each of the platforms that supports Java. That way, you can write code once and deploy it on any system that has a JVM.

  7. OK, so what happens when the compiler doesn't like what it finds in a source file? Coincidentally, a source file with compile-time errors is present in the start-here directory. Try to compile it:

    javac BadCode.java

    This time, there were compilation errors, and the Java compiler wrote error messages to the console.

  8. Why not run the CompileMe program you compiled a couple of minutes ago? Try this:

    java CompileMe

    This command starts the JVM and passes the name of the class you want to run. The suffix .class is implied. (A funny thing about Java is that the compiler requires you to include the suffix .java for the source files, but the Java runtime doesn't want to see the .class suffix for class files. Go figure.)

    So, you should have seen output similar to this:

    Exception in thread "main" java.lang.NoSuchMethodError: main

    Lovely, isn't it? You might surmise from this that the Java runtime expects to find a method named main in the class file you passed in. If you recall, there is no such method in the class CompileMe.

  9. Let's create a class that has a main method in it. Open your favorite text editor (or whatever text editor is available) and create a file named RunMe.java in the start-here directory. Give the file the following contents: 

    public class RunMe {
    public static void main(String[] args) {
        CompileMe app = new CompileMe();
        System.out.println(app.thisClassName());
    }
}

    Now compile RunMe. Go on. You know how.

    When you execute RunMe (yeah, you know how to do that, too), expect to see console output like this:

    CompileMe

    OK, so you have a working Java program. Congratulations. Looking back at the source code of CompileMe, you see that it contains a method named thisClassName. That methods does some cryptic stuff that hasn't been explained yet to get its own class name. The RunMe class then does some other cryptic stuff that results in the string CompileMe being displayed on the console.

    Whew.
