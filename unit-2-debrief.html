Unit 2: Debrief

What did we learn in Unit 2? We learned:

  • Java code is packaged into...well, into packages.
  • By convention, package names are all lower-case, with segments separated by dots.
  • The segments in package names correspond directly with directories in a directory hierarchy.
  • The Java runtime depends on the package-related directory structure to locate class files.
  • The Java runtime searches directories and jar files specified in the CLASSPATH environment variable or in a list provided with command-line argument -cp.
  • If you don't declare a package for a class, Java places the class in the default package.
  • It is a good practice to define all your classes in specific packages, and not to leave code in the default package.
  • Java provides several useful packages you can use in your programs.
  • The java.lang package contains basic Java functionality, and is always visible to your code.

There are two errors in particular that plague Java programmers. One of them is that the classpath is not set correctly. Because Java performs late binding, the compiler can't warn you about this; you only find out about it when the JVM throws exceptions at run time when it can't find a class you've told it to load. Fair warning: This will happen to you. Welcome to the club. The second common error is a topic for another unit.
