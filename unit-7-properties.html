Unit 7: Properties

It's often useful to externalize certain data so that applications can be deployed in different environments and so that various runtime configuration options can be set without having to recompile the code. In Java applications, you can use any sort of file or database to support this. The basic mechanism is to use a Java properties file, which can easily be loaded into a Properties instance by the application.

The goals of this unit are:

  1. Learn how to load a Properties instance and query the values it contains;
  2. Learn the different ways to locate a .properties file.

The first step is to set up an Eclipse/Maven project for the unit, named properties. You can copy the POM from Unit 4 and change the names of things; the structure and dependencies defined in the POM are the same for this unit.

Here is the User Story for this unit: 

  Feature:
    As a Java programmer, I want to learn how to use Java properties files
    so that I can externalize runtime data.

  Scenario: Locating and reading properties files
    Given a properties file located in the project root directory
    When I open it as a FileInputStream using a relative path
    Then I can read it and populate a Properties object

    Given a properties file located in the project root directory
    When I open it with ClassLoader.getResourceAsStream using a relative path
    Then I can read it and populate a Properties object
